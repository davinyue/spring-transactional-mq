# Kafka 发送消费支持

## 背景

当前事务消息组件已经支持 RabbitMQ 的发送、消费、自动注册和 starter 条件装配，并且多 MQ 路由能力已经具备。但 Kafka 的具体适配器尚未实现，无法验证“多 MQ 并存”在 Kafka 场景下的完整链路。

## 目标

- 参考 RabbitMQ 现有实现，新增 Kafka 的发送和消费支持。
- 保持主 starter 不变，在存在 Kafka 依赖时按条件自动装配 Kafka。
- 支持 Kafka 与 RabbitMQ 在同一应用中并存，由 `mqType` 路由发送。

## 范围

- 新增 `spring-transactional-mq-kafka` 模块。
- 实现 Kafka 生产者适配器、消费者调用器、消息监听器、自动注册器。
- 新增 Kafka starter 自动装配类，并将其注册到 `spring.factories`。
- 调整父工程和 starter 依赖管理，使 Kafka 模块可选引入、不向业务系统强制透传。
- 增加 Kafka 发送、消费、装配和多 MQ 并存相关测试。

## 非目标

- 本次不实现 Kafka 的真实环境集成测试。
- 本次不调整 `TransactionalMessageConsumer.getQueueName()` 命名，Kafka 第一版仍将其视为 topic 名。
- 本次不引入独立的 Kafka 配置项抽象层。

## 关键设计

- 新增模块 `spring-transactional-mq-kafka`，依赖 `core/api/common` 和 `spring-kafka`。
- `KafkaProducerAdapter` 实现 `MqProducerAdapter`：
  - `supportMqType() -> MqType.KAFKA`
  - `destination` 作为 topic
  - `route` 第一版作为 message key
  - 继续透传 `messageKey`、`producerCode`、`mqType`、`bizKey`、`parentId`、`rootId` 和业务 headers
- `KafkaConsumerMessageListener` 负责：
  - 反序列化 gzip 负载
  - 构造 `ConsumeContext`
  - 幂等校验
  - 调用业务消费者
  - 成功时 ack，失败或回滚时 `nack(10000L)` 触发重试
- `KafkaConsumerRegistrar` 复用自动扫描 `TransactionalMessageConsumer` 的方式，按 topic 自动创建监听容器。
- starter 新增 `TransactionalMqKafkaAutoConfiguration`，仅在 Kafka 相关类存在时装配。
- `spring-transactional-mq-spring-boot-starter` 对 Kafka 模块采用 `optional` 依赖。

## 验收标准

- `MqProducerRouter` 可根据 `MqType.KAFKA` 路由到 Kafka 生产者。
- Kafka 生产者能够按约定发送 topic/key/headers/payload。
- Kafka 消费监听器能够完成反序列化、幂等记录、业务调用和 ack/nack。
- Kafka 自动装配在依赖存在时可生效，依赖缺失时不报错。
- RabbitMQ 与 Kafka 可在同一应用中同时装配。
