# 枚举字段按下标存储

## 背景

当前数据库实体中的枚举字段通过默认方式映射，建表 SQL 中对应列仍为字符串类型。需求要求将这些枚举字段统一按枚举下标存储，并同步调整实体映射与建表脚本。

## 目标

将所有数据库实体中的枚举字段改为通过 `TypeHandler` 按枚举下标持久化，并将建表 SQL 中对应列调整为整型。

## 范围

- 调整 `spring-transactional-mq-store-ezmybatis` 模块中的数据库实体枚举字段映射
- 调整 `spring-transactional-mq-store-ezmybatis` 模块中的多数据库建表 SQL
- 增加与本次改动直接相关的最小测试

## 非目标

- 不修改业务层 record/model 中的枚举定义
- 不处理已有历史库表的数据迁移脚本
- 不改动与枚举持久化无关的仓储逻辑

## 关键设计

数据库实体中的枚举字段统一增加 `org.rdlinux.ezmybatis.annotation.TypeHandler` 注解，并指定 MyBatis 的 `EnumOrdinalTypeHandler`，使枚举按下标读写数据库。建表 SQL 中对应的 `mq_type`、`message_status`、`consume_status`、`send_status` 列统一由字符串类型改为整型类型，并保留原有索引结构不变。

## 验收标准

- 所有数据库实体中的枚举字段均显式声明按枚举下标存储的 `@TypeHandler`
- 所有建表 SQL 中的对应枚举列均为整型
- 新增测试能够校验实体注解与 SQL 列类型约束

## 风险与阻塞

枚举按下标存储会依赖枚举常量声明顺序，后续若调整枚举顺序会影响已落库数据含义。当前仅调整初始化建表 SQL，若已有存量库表和历史数据，仍需单独执行数据迁移与字段类型变更。
