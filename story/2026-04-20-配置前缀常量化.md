# 配置前缀常量化

## 背景

当前 starter 中多处直接使用 `"transactionalmq"` 字符串作为配置前缀，后续如果需要调整前缀，容易出现遗漏。

## 目标

- 将 `transactionalmq` 配置前缀定义为统一常量。
- 自动装配注解和测试代码复用该常量。

## 范围

- 调整 starter 配置类和自动配置类中的配置前缀引用。
- 调整 starter 相关测试中的配置项拼接。

## 非目标

- 本次不修改实际配置前缀值。
- 本次不调整 README 中面向用户展示的配置示例。

## 关键设计

- 在 `TransactionalMqProperties` 中定义 `public static final String PREFIX = "transactionalmq"`。
- `@ConfigurationProperties` 和 `@ConditionalOnProperty` 统一引用该常量。
- 测试中的配置项使用 `TransactionalMqProperties.PREFIX + ".xxx"` 拼接。

## 验收标准

- main 代码中不再直接硬编码 `"transactionalmq"` 作为配置前缀。
- starter 测试通过。
