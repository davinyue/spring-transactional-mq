# 自动建表初始化

## 背景

当前事务消息组件虽然提供了多数据库建表 SQL，但不会在启动时根据 ez-mybatis 的数据库类型自动选择并执行脚本。业务接入时仍需要手工配置 `schema-locations` 或自行执行建表语句，接入成本较高。

## 目标

- 在框架启动时默认自动尝试建表。
- 基于 ez-mybatis 的数据库类型动态选择匹配的建表 SQL。
- 建表失败时不影响应用启动。

## 范围

- 为 starter 增加自动建表开关配置，默认开启。
- 增加 schema initializer，根据 `EzMybatisContent.getDbType(Configuration)` 选择 SQL。
- 增加相关单元测试和自动装配测试。
- 更新 README 中的接入说明。

## 非目标

- 本次不实现跨数据库的表存在性检查。
- 本次不记录建表失败日志。
- 本次不修改现有仓储与消息收发主流程。

## 关键设计

- 在 `transactionalmq` 配置下新增 `auto-init-schema`，默认 `true`。
- 启动时从 MyBatis `Configuration` 中读取 ez-mybatis `DbType`。
- 按 `DbType` 选择 `spring-transactional-mq-store-ezmybatis` 中已有 SQL 脚本。
- 执行脚本时捕获所有异常并直接吞掉，不影响启动。

## 验收标准

- 默认启动时会尝试自动建表。
- `transactionalmq.auto-init-schema=false` 时不执行自动建表。
- 能根据 ez-mybatis 数据库类型选择正确脚本。
- 建表异常不会导致 Spring 容器启动失败。
