# 统一消息发送目标抽象

## 背景

当前发送目标主要依赖 RabbitMQ 的 `destination=exchange:routingKey` 约定，后续接入 RocketMQ、Kafka 时会被 RabbitMQ 概念绑定。

## 需求

- 核心 API 使用通用字段表达发送目标，兼容 RabbitMQ、RocketMQ、Kafka。
- 保留现有 RabbitMQ `destination=exchange:routingKey` 旧写法兼容。
- RabbitMQ 新写法使用 `destination` 表示 exchange/queue，`route` 表示 routingKey。
- 增加 `shardingKey`，为 RocketMQ queueSelector 与 Kafka partition key 预留。
- 持久化实体与建表 SQL 同步增加必要字段。
