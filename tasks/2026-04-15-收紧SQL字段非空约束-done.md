# 收紧 SQL 字段非空约束

## 背景

`create_time`、`update_time` 作为审计字段不应为空；部分业务时间和默认值字段也应在建表时声明非空，避免脏数据进入核心表。

## 需求

- 所有表的 `create_time`、`update_time` 改为 `NOT NULL`。
- 重新判断其它字段是否允许为空，必要字段改为 `NOT NULL`。
- 补齐写入路径中的默认审计时间，避免非空约束导致插入失败。
- 更新所有数据库建表 SQL。

## 字段判断

- `route`、`sharding_key`、`biz_key`、`next_dispatch_time`、`description` 可为空。
- `payload_text`、`headers_json`、`consume_time`、`archive_time`、`retry_count`、`last_send_time` 不应为空。
- `message_status`、`consume_status`、`send_status` 已为非空，写入映射继续提供默认状态兜底。
